# rokudev-tools v0.7.2 brs-docs wheel-bundle fix

**Date:** 2026-05-20
**Type:** Patch release. Single bug fix on top of v0.7.1.

## Bug fix

**Wheel installs (`pip install brs-docs`, `uvx brs-docs`) shipped a
27 KB wheel with no bundled corpus.**

`src/brs_docs/data/corpus.sqlite` is gitignored (it's a build artifact
regenerated by `make build-corpus`). Hatch's wheel target honors
`.gitignore` by default, and the v0.7.0 / v0.7.1 pyproject.toml only
force-included `corpus.lock`. Net effect: the wheel landed without
`corpus.sqlite`, and `first_run.py` could not satisfy
`importlib.resources.files("brs_docs").joinpath("data/corpus.sqlite")`.
The first invocation of any tool (search, list, recommend, etc.) would
raise `CorpusNotInitialized` with a message asking the user to run
`make build-corpus`, which is impossible from a PyPI install.

Dev / editable / source installs (`uv run`) were unaffected because
`importlib.resources` resolves against the live src tree.

## Why the test suite didn't catch it

No test built the wheel and inspected its contents. The corpus-build
tests only verified the dev-tree path.

## Changes

- `packages/brs-docs/pyproject.toml`: add
  `src/brs_docs/data/corpus.sqlite` to the wheel `force-include` map
  alongside the existing `corpus.lock` entry.
- `packages/brs-docs/tests/contract/test_wheel_bundles_corpus_sqlite.py`:
  new integration test that subprocesses `uv build --wheel`, unzips
  the result, and asserts `brs_docs/data/corpus.sqlite` is present and
  the wheel is >1 MB. Skips gracefully if `make build-corpus` has not
  been run.
- Version bump `__version__` + `pyproject.toml` + `corpus.lock`
  `brs_docs_version` 0.7.1 → 0.7.2.

## Cumulative status of post-T29 verification

- v0.7.0 shipped with TWO latent bugs (companion-lock name, wheel
  missing sqlite).
- v0.7.1 fixed the companion-lock name; dev/source installs work.
- v0.7.2 fixes the wheel bundle; pip / uvx installs work.

Both bugs were uncovered by the spec-mandated post-T29 success-criteria
verification checklist. Lesson for future: every step that says "wheel
ships X" needs a wheel-inspection contract test, not just a source-tree
test.
